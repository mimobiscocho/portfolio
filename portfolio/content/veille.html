Sécurité des hyperviseurs de virtualisation

Veille Technologique — BTS SIO SLAM 2025

Les hyperviseurs (Proxmox VE, VMware ESXi, KVM) sont devenus des cibles prioritaires pour les attaquants : compromettre l'hyperviseur, c'est prendre le contrôle de toutes les VMs hébergées en un seul coup.

VM Escape : faille permettant à une VM de s'échapper vers l'hôte — CVE-2023-20867 (VMware Tools) exploitée in the wild en 2023.

Proxmox & attaque API : l'API REST sans MFA est vecteur d'intrusion — +12 000 instances exposées sur Shodan en 2024.

Ransomware ESXiArgs : chiffrement massif de serveurs ESXi via CVE-2021-21974, exploitée 2 ans après le patch.

Bonnes pratiques : VLAN management isolé, accès VPN uniquement, MFA sur interface web, snapshots chiffrés, mises à jour immédiates.

Dans un contexte de virtualisation massive, sécuriser la couche hyperviseur est aussi critique que le périmètre réseau. Une bonne hygiène de configuration suffit à éliminer la majorité des vecteurs documentés.
